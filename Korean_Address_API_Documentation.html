1. Overview

This project provides a Korean address search service by proxying the juso.go.kr (주소기반산업지원서비스) Open API from the Korean Ministry of Interior and Safety.

It enables searching Korean addresses by keyword and returns structured address data including postal code, road address, lot number address, city, district, and building information.

Key Features

  • Search Korean addresses by keyword (Korean or English)
  • Server-side proxy to juso.go.kr (avoids CORS issues)
  • Pagination support
  • Map view with pin marker (Leaflet + OpenStreetMap)
  • Merchant signup form integration
  • Embeddable widget for any project

2. Architecture & Flow

User Browser
│
▼ Click "Find Address" → Search keyword
│
ASP.NET Backend (localhost:5600)
GET /api/address?keyword=...&language=kr
│
▼ Proxy request with confmKey
│
juso.go.kr API
POST addrLinkApi.do?keyword=...&confmKey=...
│
▼ Returns JSON (Juso list)
│
User selects address → Form fills + Map shows location

Why a Backend Proxy?

  • CORS: juso.go.kr blocks direct browser requests from non-registered domains
  • Key Security: API key (confmKey) stays on server, not exposed in frontend
  • Domain Binding: The confmKey is validated against the calling domain — server-to-server calls bypass this

4. Getting API Key (confmKey)

Step-by-Step Registration

  1. Go to https://business.juso.go.kr
  2. Click 회원가입 (Sign Up) → Create account
  3. Login → Go to 주소제공 → Open API 신청
  4. Fill the application form:
    FieldValue
    서비스명 (Service Name)Your project name
    서비스URL (Service URL)http://localhost (for development)
    활용용도 (Purpose)Address search
  5. Submit → You'll receive a confmKey (usually instant for dev)
Important: The confmKey is locked to the registered domain. If you register http://localhost, it only works from localhost. For production, update the registered URL to your production domain.

8. Backend Implementation (ASP.NET)

AddressService.cs — Core Logic

public async Task<JusoApiResponse> SearchAsync(
    string keyword, int page, int count,
    string language, string? confmKey = null)
{
    // Select URL based on language
    string baseUrl = language == "en" ? _config.EnUrl : _config.KrUrl;

    // Use provided key or fall back to config
    string key = string.IsNullOrEmpty(confmKey)
        ? (language == "en" ? _config.EnKey : _config.KrKey)
        : confmKey;

    // Build query string
    var query = HttpUtility.ParseQueryString(string.Empty);
    query["currentPage"] = page.ToString();
    query["countPerPage"] = count.ToString();
    query["keyword"] = keyword;
    query["confmKey"] = key;
    query["resultType"] = "json";

    // Call juso.go.kr
    var response = await _httpClient.PostAsync(
        $"{baseUrl}?{query}", null);
    var body = await response.Content.ReadAsStringAsync();

    return JsonSerializer.Deserialize<JusoApiResponse>(body);
}

Program.cs — Endpoint Registration

// Configure DI
builder.Services.Configure<AddressApiConfig>(
    builder.Configuration.GetSection("AddressApi"));
builder.Services.AddHttpClient<AddressService>();

// Search endpoint
app.MapGet("/api/address", async (
    string keyword, int page, int count,
    string language, string? confmKey,
    AddressService addressService) =>
{
    var result = await addressService.SearchAsync(
        keyword, page, count, language, confmKey);
    return Results.Ok(result);
});

10. Embeddable Widget

Use the address search dialog in any HTML page with just 2 lines:

Step 1: Include the Script

<script src="http://localhost:5600/js/address-widget.js"></script>

Step 2: Open the Dialog

function findAddress() {
    AddressWidget.open({
        apiUrl: 'http://localhost:5600/api/address',
        language: 'kr',   // 'kr' or 'en'
        onSelect: function(addr) {
            // addr = full Juso object
            document.getElementById('zip').value = addr.zipNo;
            document.getElementById('addr').value = addr.roadAddr;
            document.getElementById('city').value = addr.siNm;
        }
    });
}
The widget is self-contained: It injects its own CSS, creates the dialog DOM, handles search, pagination, and selection — no dependencies required.

11. Map Integration (Leaflet)

After address selection, a map is shown using Leaflet.js + OpenStreetMap tiles with geocoding via Nominatim.

How It Works

  1. User selects an address from search results
  2. Frontend sends the road address to Nominatim geocoding API
  3. Receives latitude/longitude coordinates
  4. Renders Leaflet map with pin marker and popup
  5. Shows "Open in Google Maps" link

Dependencies (included via CDN)

<!-- In index.html head -->
<link rel="stylesheet" href="https://unpkg.com/leaflet@1.9.4/dist/leaflet.css" />
<script src="https://unpkg.com/leaflet@1.9.4/dist/leaflet.js"></script>

Geocoding (Nominatim — Free, No API Key)

const query = encodeURIComponent(addr.roadAddr);
const res = await fetch(
    `https://nominatim.openstreetmap.org/search?q=${query}&format=json&countrycodes=kr&limit=1`
);
const geo = await res.json();
const lat = parseFloat(geo[0].lat);
const lng = parseFloat(geo[0].lon);
Nominatim Usage Policy: Max 1 request/second, include a User-Agent header for production use. For high-volume, consider self-hosting Nominatim or using a paid geocoding service.
